# Night review checklist

Work through these in order. Use only the **rollup and excerpts** provided in the job context — do not invent events you cannot see.

1. **Volume** — How active was the day (messages, chats)? Anything unusual?
2. **Patterns** — Recurring themes, stuck topics, or repeated questions?
3. **Tools** — Which tools appeared (from the rollup)? Failures, missing steps, or obvious gaps where a new capability would help?
4. **Memory** — Do recent memories align with chat themes? Anything important missing from memory that should be suggested?
5. **Improvements** — Concrete, actionable tweaks (prompt habits, reminders, follow-ups) — not generic praise.

If there is **nothing material** to report after an honest scan, respond with structured output where `status` is `ok` and arrays are empty (equivalent intent to OpenClaw’s `HEARTBEAT_OK`).

Do not include API keys, passwords, or private contact details in any field.
